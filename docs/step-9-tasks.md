# Step 9 — Build Pipeline

Each package currently resolves via TypeScript source files through workspace
symlinks — correct for development, not suitable for distribution. This step
produces `dist/` output per package and verifies the built artifacts are
correct. Publishing to npm is deferred to after Step 10 (documentation).

---

## Decisions (already confirmed)

| Decision | Choice |
|---|---|
| Build tool | `tsup` |
| Output format | ESM-only (`format: ['esm']`) |
| Type declarations | `dts: true` on each tsup config |
| Version | `0.3.0` lockstep (already bumped in B4) |
| `exports` in dev | `./src/index.ts` (unchanged — tests keep working) |
| `exports` at publish | `publishConfig.exports` → `dist/index.js` + `dist/index.d.ts` (already in B2) |
| WASM binary delivery | External file on disk — `wasm/pkg/` included in `files` (already in B1) |
| Publish strategy | Each package published independently to npm under `@paragraf/` scope |

## Phase S1 — Install tsup

Single dev-dependency at the monorepo root. All packages share it via
workspace hoisting.

```bash
npm install --save-dev tsup --workspace-root
```

---

## Phase S2 — Add `tsup.config.ts` to each package

All packages share the same base shape. Special cases noted.

### Base config (applies to 6 of 8 packages)

```ts
// tsup.config.ts
import { defineConfig } from 'tsup';

export default defineConfig({
  entry: ['src/index.ts'],
  format: ['esm'],
  dts: true,
  clean: true,
  outDir: 'dist',
  sourcemap: false,
});
```

Applies to: `0-types`, `1a-linebreak`, `1b-font-engine`, `2b-render-core`,
`3a-typography`, `3b-render-pdf`, `0-color`.

### `2a-shaping-wasm` — special config

The wasm-bindgen glue (`wasm/pkg/knuth_plass_wasm.js`) must not be bundled —
it uses `require('fs').readFileSync` and `__dirname` to locate the `.wasm`
file at runtime. Bundling would break the path.

```ts
// tsup.config.ts
import { defineConfig } from 'tsup';

export default defineConfig({
  entry: ['src/index.ts'],
  format: ['esm'],
  dts: true,
  clean: true,
  outDir: 'dist',
  sourcemap: false,
  external: ['../wasm/pkg/knuth_plass_wasm.js'],
  noExternal: [],
});
```

The relative path `../wasm/pkg/knuth_plass_wasm.js` is correct from `dist/`
because `wasm/pkg/` sits at the same level as `dist/` inside the package root.

## Phase S3 — Add `build` script to each package `package.json`

Add `"build": "tsup"` to `scripts` in all 8 packages.

Add `"build": "npm run build --workspaces --if-present"` to the root
`package.json` so one command builds everything in the correct order
(npm respects workspace dependency graph).

---

## Phase S4 — Verify each build output

After running `npm run build` from the monorepo root, confirm:

- [ ] `dist/index.js` exists in each package
- [ ] `dist/index.d.ts` exists in each package
- [ ] `2a-shaping-wasm/dist/index.js` does **not** inline `knuth_plass_wasm.js`
      (grep for `readFileSync` — it must be absent from the bundle)
- [ ] `2a-shaping-wasm/wasm/pkg/knuth_plass_wasm_bg.wasm` is still a separate file
- [ ] `npm run test` still passes (sources still resolve via workspace symlinks
      to `src/`, not `dist/`)

---

## Phase S5 — Smoke-test the dist output

Verify the built output is importable and contains the right files.

### Tarball contents check

```bash
cd 0-types && npm pack --dry-run
cd 1a-linebreak && npm pack --dry-run
# ... repeat for all 8
```

Confirm per package:
- [ ] `dist/` is included
- [ ] `src/`, `tests/`, `tsconfig.json`, `tsup.config.ts` are excluded
- [ ] `2a-shaping-wasm` includes `wasm/pkg/` (the WASM binary and glue)

Note: `2a-shaping-wasm/wasm/pkg/.gitignore` contains `*` (generated by
wasm-pack). Work around this by adding a `wasm/pkg/.npmignore` that
overrides it, allowing `npm pack` to include the pre-built artifacts.

### Import smoke test (optional)

A minimal `scripts/smoke-dist.ts` that temporarily imports from `dist/`
to confirm the ESM output actually loads and exports the expected symbols.
Defer this if the tarball check is sufficient.
